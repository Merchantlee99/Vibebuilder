# Visual Evidence

Store generated screenshots, accessibility exports, layout maps, and visual reports here when a project adopts the v5 harness.

Rules:

- Prefer `harness/evidence/artifacts/<task-id>/` for task-scoped deliverables.
- Keep long-lived visual baselines only when the target app has stable data and deterministic rendering.
- Redact or avoid screenshots for sensitive screens.
- Pair screenshots with DOM/layout/accessibility evidence for `normal+` UI work.
