# Browser QA Playbook

## Browser First

- 공식 문서 확인
- 릴리스 노트 확인
- 웹 화면 QA
- 인용이 필요한 조사

## QA Loop

1. 목표 흐름 한 줄 정의
2. 핵심 경로 실행
3. 실패 상태와 빈 상태 확인
4. 콘솔 또는 네트워크 이상 징후 기록
5. `Documentation.md`와 review artifact에 결과 반영

## Escalate to Computer Use When

- 로그인이나 2차 인증이 브라우저 흐름을 막을 때
- 로컬 앱 조작이 필요할 때
- 웹 도구만으로 재현이 안 될 때
