# Project Profile

## Type

Codex-native harness template for solo or small-team vibecoding projects.

## Operating Assumptions

- Primary operator is a single user driving Codex app directly.
- Codex subagents are allowed and are expected to be useful.
- Long-running work should prefer explicit documentation and automation over chat memory.
- Worktree isolation is preferred for non-trivial write tasks.

## Initial Style

- Primary language for operator-facing docs: Korean.
- Operational docs should stay short, explicit, and restart-friendly.
- Scripts should use Python stdlib where possible.
