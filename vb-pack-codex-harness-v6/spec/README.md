# Spec Layer

Codex Harness v6 uses `spec/` as the governing layer for accepted intent.

Minimum shape:

```text
spec/
  00_constitution.md
  features/<feature>/spec.md
  reviews/
  changes/
```

Use `docs/ai/current/Feature-Spec.md` for active task work, then promote accepted behavior into `spec/features/<feature>/spec.md` when the project adopts the template.
